->Enabling secure communication
Last Updated: 2024-05-10
You can enable Debug Profile Service to communicate via a Secure Sockets Layer (SSL) certificate with a private key and self-signed certificate stored in a keystore file or via a Certificate Authority (CA) certificate with a RACF managed key ring.

Remote clients such as the z/OS Debugger Profiles view might perform hostname verification when communicating with Debug Profile Service. Make sure that the hostname is defined in the Subject Alternative Name (SAN) extension of the SSL certificate.

Enabling secure communication with a keystore file
You can use the sample keystore file installed in /etc/debug/keystore.p12, or create a new one with the following steps:

Create a keystore file by using the Java runtime utility keytool:
keytool -genkey -alias liberty -keyalg RSA -storetype PKCS12 -keystore keystore.p12 -ext "SAN=DNS:`/bin/hostname`"

Edit /etc/debug/eqaprof.env and update the keystore variables to use the new keystore file.
To ensure that the keystore file is only readable by the protected user ID STCEQA, change the owner and permission of the file with:
  chown STCEQA:STCGROUP keystore.p12
  chmod 640 keystore.p12

Export the SSL certificate with:
keytool -export -keystore keystore.p12 -alias liberty -storetype PKCS12 -file liberty.cer -rfc

The SSL certificate can be distributed to remote users to be imported into the client keystore.
For Eclipse users, manually importing the certificate for z/OS Debugger Profiles view users is not required because users will be prompted to accept the certificate if it is not already in the keystore the first time the view connects to Debug Profile Service.

Z Open Debug users still need to manually import the self-signed certificate.

Configuration* ObjXfer::postConfiguration(const string &url, Configuration* cfg, Logger* log)
{
    struct ObjectReceived objRecv;
    std::string serializedConfig;
    if(!cfg->SerializeToString(&serializedConfig)) {
        if(log) log->error("Failed to serialize Configuration protobuf");
        return nullptr;
    }

    curl_easy_setopt(pcurl_, CURLOPT_URL, url.c_str());
    curl_easy_setopt(pcurl_, CURLOPT_VERBOSE, 0);
    curl_easy_setopt(pcurl_, CURLOPT_WRITEFUNCTION, record_write);
    curl_easy_setopt(pcurl_, CURLOPT_WRITEDATA, &objRecv);
    curl_easy_setopt(pcurl_, CURLOPT_NOSIGNAL, 1L);
    curl_easy_setopt(pcurl_, CURLOPT_CONNECTTIMEOUT, getConnTimeout());

    curl_easy_setopt(pcurl_, CURLOPT_POST, 1L);
    curl_easy_setopt(pcurl_, CURLOPT_POSTFIELDS, serializedConfig.c_str());
    curl_easy_setopt(pcurl_, CURLOPT_POSTFIELDSIZE, serializedConfig.size());

    struct curl_slist* http_headers = nullptr;
    http_headers = curl_slist_append(http_headers, "Content-Type: application/x-protobuf");
    std::string authHeader = "Authorization: Bearer " + accessToken; // use your static token
    http_headers = curl_slist_append(http_headers, authHeader.c_str());
    curl_easy_setopt(pcurl_, CURLOPT_HTTPHEADER, http_headers);

    CURLcode res = curl_easy_perform(pcurl_);
    curl_slist_free_all(http_headers);

    if(res != CURLE_OK) {
        if(log) log->error("Failed POST: %s", curl_easy_strerror(res));
        return nullptr;
    }

    Configuration* respCfg = new Configuration();
    if(!respCfg->ParseFromString(objRecv.response)) {
        if(log) log->error("Failed to parse POST response");
        DELETE(respCfg);
        return nullptr;
    }

    return respCfg;
}
